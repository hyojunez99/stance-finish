# 쇼핑몰 프론트엔드 웹 앱

## 1. 프로젝트 소개

- **설명:**  
러닝 액티브웨어 쇼핑몰 SPA로, React 기반으로 Mock JSON 데이터를 활용해 상품/뉴스/카테고리 UI 및 스크롤 애니메이션을 구현한 커머스 프론트엔드 웹 앱

- **진행 기간:** 2025.12.19 ~ 2025.12.29 (9일)  
- **개발 인원:** 5인 팀 프로젝트  
- **배포 링크:** [Stance-Finish](https://hyojunez99.github.io/stance-finish/)  
- **GitHub:** [Repository Link](https://github.com/hyojunez99/stance-finish.git)  

---

## 2. 사용 기술 스택 (Tech Stack)

- **Frontend:** JavaScript (ES6+), React.js, SCSS  
- **Data Handling:** Mock JSON  
- **Tools:** Figma, Git/GitHub, GSAP  

---

## 3. 기획 및 디자인 (Planning & Design)

- **컨셉:**  
사용자 친화적인 쇼핑몰 SPA로, 상품 탐색과 구매 흐름을 직관적으로 경험할 수 있도록 UI/UX 설계  

- **Tool:** Figma  
- **주요 활동:**
  - JSON 데이터 및 이미지 구조 관리  
  - 메인 페이지 UI 구현  
  - 장바구니 페이지 UI 기획 및 와이어프레임 제작  

---

## 4. 디렉토리 구조

```text
src
│
├── assets # 이미지, 폰트 및 JSON 데이터
│ │
│ ├── data # 화면에 쓰이는 고정 데이터 관리
│ │
│ ├── images # 이미지 파일 보관
│ │
│ └── scss # 공통으로 쓰이는 폰토, 색상, 반응형 관리 폴더
│
├── components # 각 페이지와 공통 영역을 구성하는 컴포넌트를 관리 폴더
│ │
│ ├── common # 여러 페이지에서 공통으로 재사용되는 컴포넌트
│ │
│ ├── detail # 상품 상세 페이지 전용 컴포넌트 폴더
│ │
│ └── main # 메인 페이지(홈) 전용 컴포넌트 폴더
│
├── layout # 공통 레이아웃 구조 (Header, Footer )
│  
└── pages # 라우팅 페이지 (Cart, Detail, Main...)

```

## 7. 트러블 슈팅

- **문제 1:** 초기 단계에서 데이터 흐름 합의 부족으로 팀원마다 컴포넌트 구조와 JSON 사용 방식이 다르게 구현됨
  - **해결:** 디자인 시안을 기준으로 섹션별 데이터 단위를 정리하고 공통 JSON 구조를 정의 후 팀원들과 공유

- **문제 2:** Slack 사용 중 일부 기능 관련 정보가 Figma 내부에만 공유되어 팀원 간 정보 전달 누락
  - **해결:** 기능 구현 관련 논의는 Slack 채널에서 공유하고, Figma는 디자인 참고용으로만 사용하도록 협업 방식 개선
    
- **문제 3:** 데스크탑 기준 레이아웃을 모바일에 적용할 때 상품 카드 크기와 섹션 간 여백이 어색
  - **해결:** 화면 크기별 레이아웃 재점검, Grid와 Flex를 분리 적용, 여백과 카드 비율 조정으로 반응형 완성도 향상
