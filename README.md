## 7. 트러블 슈팅

문제 1 : 데이터 흐름에 대한 합의가 없어 컴포넌트 구조와 데이터 사용 방식이 서로 다르게 구현되었습니다.

해결 : 디자인 시안을 다시 검토하며 섹션별 데이터 단위 방식을 정리하고, 공통으로 사용할 JSON 데이터 구조를 정의해 공유했습니다.

문제 2 : 여러개의 JSON데이터를 쓰다보니 화면에 구현이 잘 안됨

해결 : 각 섹션이 필요한 데이터만 따로 관리하도록 구조를 정리해 수정했습니다.

개발 후 느낀점 
팀 프로젝트를 진행하며 구현하는 것 자체보다 데이터 구조를 어떻게 설계하고 공유하느냐가 개발의 더 큰 영향을 준다는 점을 느꼈습니다. 또한 여러 JSON 데이터를 다루며 필요한 데이터만 관리하는 구조의 중요성을 느겼고, 앞으로는 기능 구현 전에 데이터 설계와 역할 분담을 먼저 명확히 하는 개발 방식이 필요하다는 것을 배우는 계기가 되었습니다.
